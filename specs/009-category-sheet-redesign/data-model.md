# Data Model: Category Sheet Redesign

**Branch**: `009-category-sheet-redesign` | **Date**: 2026-03-27

## Overview

This feature introduces **no database schema changes**. The `Category` entity and `AllocationBucket` type already exist in `web/types/index.ts` and are inferred from the Drizzle schema. No migrations are required.

---

## Existing Entity: Category

Defined in `web/types/index.ts`. Unchanged by this feature.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `string` | PK, UUID | Generated by database |
| `userId` | `string` | FK → user, required | All queries scoped by this field |
| `name` | `string` | min 1, max 100 chars | Validated by `createCategorySchema` |
| `type` | `"expense" \| "income"` | required | Determines bucket visibility |
| `allocationBucket` | `"needs" \| "wants" \| "future" \| null` | required if type=expense | Null for income categories |
| `icon` | `string \| null` | max 10 chars (emoji) | Optional; falls back to default `💡` in UI |
| `createdAt` | `Date` | auto | Set on insert |
| `updatedAt` | `Date` | auto | Set on update |

**Uniqueness**: No uniqueness constraint on `name` at the database level. Duplicate names within the same user+type are permitted. Server-side errors (e.g., unexpected DB errors) surface through the `ActionResult` type.

## New Client-Side Constants (no persistence)

These are UI-only constants added to `web/components/categories-page.tsx`. They have no database representation.

### `BUCKET_PILLS`

| Field | Value |
|-------|-------|
| `key` | `"needs" \| "wants" \| "future"` |
| `label` | `"Needs" \| "Wants" \| "Future"` |
| `Icon` | `Home \| Coffee \| PiggyBank` (lucide-react) |

### `CATEGORY_EMOJIS`

A fixed array of 30 emoji strings used to populate the icon grid picker. No persistence — selected emoji is stored in `CategoryFormState.icon` (component state) and submitted to the `createCategory` / `updateCategory` server action as the `icon` field.

---

## Form State Shape

The `CategoryFormState` type (internal to `categories-page.tsx`) maps directly to server action inputs:

| Field | Type | Maps to |
|-------|------|---------|
| `name` | `string` | `CreateCategoryInput.name` / `UpdateCategoryInput.name` |
| `icon` | `string` | `CreateCategoryInput.icon` / `UpdateCategoryInput.icon` |
| `type` | `TransactionType` | `CreateCategoryInput.type` |
| `allocationBucket` | `AllocationBucket \| ""` | `CreateCategoryInput.allocationBucket` / `UpdateCategoryInput.allocationBucket` |

Empty string `""` for `allocationBucket` is used as the unset sentinel in local state; it is converted to `undefined` before submission so it does not fail Zod validation.

---

## Validation Rules (unchanged)

Defined in `web/lib/validations/categories.ts`:

- `name`: required, 1–100 characters
- `icon`: optional, max 10 characters
- `type`: `"expense" | "income"`
- `allocationBucket`: required when `type === "expense"`, forbidden for `income`

No changes to validation schemas in this feature.
